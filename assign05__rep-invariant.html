SWE 419 Assignment 5
Spring 2025

Goal: JUnit, Abstraction Function, and Rep-Invariants

  1. First you need to learn how to write JUnit tests (go over the JUnit tutorials). You are provided with the following files to help you write tests and run them: run.bat, hamcrest-core-1.3.jar, junit-4.13.1.jar, AllTests.java, BoundedQueueTester.java, BoundedQueueBuggy1Tester.java, BoundedQueueBuggy2Tester.java.
  2. Implement the provided rep-invariant in "boolean repOK()". Also implement the Abstraction Function for class BoundedQueue. Recall that in Java the AF is implemented as the "String toString()" method. The squeletons for repOk() and toString() are provided.
  3. After implementing repOk() and toString(), add the following tests to BoundedQueueTester: a) one happy-path test for each of isEmpty(), isFull(), put(), get(), putAll(), getAll(); b) one test that throws an exception for each of put(), get(), putAll(), getAll(); c) at least five tests for your implementation of toString(). Note that the six tests in a) should additionally include the following assertion at their respective ends "assertTrue(someBoundedQueue.repOk()), to ensure that the repOK() was not violated.
  4. Create a duplicate of class BoundedQueue, called BoundedQueueBuggy1. Inject a bug in BoundedQueueBuggy1, e.g., delete a statement or change/add an operator etc. Your injected bug should violate the rep-invariant. Add comments in BoundedQueueBuggy1 identifying your injected bug.
  5. Write one or more tests in BoundedQueueBuggy1Tester that will exercise your injected bug and violate repOk().
  6. Repeat the last 2 steps by creating BoundedQueueBuggy2 and BoundedQueueBuggy2Tester.
